# 骑行能量站 - 需求分析文档

## 一、项目背景

### 1.1 问题定义
共享单车用户在骑行后缺乏有效的行为反馈与激励机制，导致：
- 用户骑行频次低，粘性不足
- 缺乏持续使用的动力
- 骑行行为价值感知不明显
- 用户流失率高

### 1.2 目标用户
- **主要用户**：18-35岁城市通勤人群
- **次要用户**：健身爱好者、环保倡导者
- **潜在用户**：偶尔使用共享单车的城市居民

### 1.3 产品定位
一款基于美团共享单车的完整游戏化骑行记录与激励工具，通过构建统一的成长体系、探索系统和社交社区，将共享单车出行转化为可感知、可积累、可分享的游戏化体验，深度融合美团生态，提升用户骑行频次与平台粘性。

**核心理念**：
- 不是分散的激励功能堆砌，而是一个完整的游戏化世界
- 以"城市探索者"为核心身份，通过骑行解锁城市、收集成就、培养伙伴
- 将骑行数据转化为游戏进度，让每次出行都有意义
- 构建骑行社区，让用户分享路线、参与挑战、互相激励

## 二、竞品分析

### 2.1 共享单车平台竞品分析

#### 2.1.1 哈啰出行（深度分析）

**核心游戏化功能**：

**1. 挑战赛系统**
- IP联动挑战（如与《头脑特工队2》合作）
- 0碳骑行挑战赛（减碳主题）
- 周周骑挑战（持续激励）
- 奖励：虚拟勋章、实体奖牌、电影票、周边

**2. 个人碳账户系统**
- 碳积分"小蓝C"
- 骑行里程转化为碳积分
- 积分兑换骑行卡、优惠券
- 参与公益活动

**3. 打卡地标活动**
- 特定地标打卡获得奖励
- 地标收集机制

**4. 福利中心任务系统**
- 做任务赚奖励金
- 观看激励视频获得随机奖励
- 积分兑换系统

**5. 游戏联动**
- 与游戏IP合作（如"无限暖暖"）
- 骑车拿游戏奖励
- VR骑行游戏体验

**优势**：
- 游戏化元素丰富多样
- 挑战赛机制成熟
- IP联动吸引力强
- 碳积分系统有社会价值
- 支付宝生态加持

**劣势**：
- **功能分散**：各个激励机制相对独立，缺乏统一的游戏世界观
- **缺乏核心成长线**：没有清晰的角色成长或养成系统
- **社区属性弱**：缺少路线分享和用户互动社区
- **生态联动有限**：主要依赖支付宝，跨业务联动不足

#### 2.1.2 青桔单车（滴滴）

**核心功能**：
- 多种骑行卡（时长卡、次卡）
- 推荐好友奖励机制
- 任务中心系统
- 滴滴生态整合

**激励机制**：
- 新用户福利：7天免费骑行卡、优惠券
- 推荐奖励：邀请好友获得现金奖励
- 任务中心：完成任务获得权益
- 滴滴会员体系（V7/V8会员免费骑行）

**优势**：
- 滴滴生态强大，出行场景全覆盖
- 会员体系成熟，跨界合作丰富（酒店、餐饮）
- 推荐机制激励力度大

**劣势**：
- 游戏化元素极少，偏向功能性激励
- 缺乏趣味性和情感连接
- 无社区和社交属性

#### 2.1.3 美团共享单车（现状）

**核心功能**：
- 车辆硬件优秀：北斗+GPS双频定位、4G智能锁、减震、手机支架
- 舒适性领先：云感透气座垫、轻量化车身
- 六级会员体系：普通到黑钻
- 神券系统：联动外卖、酒旅、电影、医药

**激励机制**：
- 新用户7天免费骑行卡
- 会员成长值系统
- 神券联动全业务

**优势**：
- **美团生态最强大**：外卖、酒旅、电影、医药全覆盖
- 会员体系成熟，神券联动性强
- 车辆硬件体验行业领先

**劣势**：
- **完全缺乏游戏化**：无任何游戏化激励机制
- 骑行行为与会员成长关联极弱
- 无社区、无社交、无挑战
- 缺乏趣味性和情感连接
- 用户粘性完全依赖硬件体验和价格

### 2.2 竞品分析总结

#### 核心发现

**哈啰的启示**：
1. 挑战赛机制有效（IP联动、主题挑战）
2. 碳积分系统有社会价值认同
3. 虚拟+实体奖励结合效果好
4. 但功能过于分散，缺乏统一体验

**美团的机会**：
1. **游戏化完全空白**：可以从零构建完整的游戏化世界
2. **生态优势巨大**：外卖、酒旅联动是独特护城河
3. **硬件体验领先**：可以在体验基础上叠加游戏化
4. **会员体系成熟**：可以深度整合骑行与会员成长

**设计方向**：
- 构建**统一的游戏化世界观**，而非分散功能
- 建立**核心成长线**（角色/伙伴养成）
- 打造**路线探索与分享社区**
- 设计**挑战赛系统**（季度主题、城市PK）
- 深度融合**美团生态**（但不过度商业化）

## 三、完整游戏化系统设计

### 3.1 核心设计理念

**世界观**：城市探索者的骑行冒险
- 用户化身"城市探索者"，通过骑行解锁城市的秘密
- 培养专属骑行伙伴（虚拟生物），共同成长
- 探索特色路线，收集城市记忆
- 参与挑战赛，与全国骑友竞技
- 分享路线故事，建立骑行社区

**统一性原则**：
- 所有功能围绕"探索-成长-分享"三大核心展开
- 能量系统是唯一的游戏货币
- 所有激励最终服务于角色成长和伙伴养成
- 美团生态奖励作为额外福利，不喧宾夺主

### 3.2 核心系统架构

### 3.3 详细功能设计

#### 【基础层】骑行记录系统

这是产品的基础功能，提供完善的骑行数据记录和可视化。

##### 3.3.0 骑行数据记录与可视化

**功能描述**：
自动记录每次骑行的详细数据，并提供丰富的可视化展示。

**单次骑行记录**：
- 骑行轨迹地图（GPS路线）
- 骑行距离（精确到0.01km）
- 骑行时长（分钟）
- 平均速度（km/h）
- 消耗卡路里（kcal）
- 减少碳排放（kg CO2）
- 起点和终点位置
- 骑行时间（开始-结束）
- 经过的区域
- 天气状况

**骑行历史**：
- 按时间排序的骑行记录列表
- 每条记录显示：日期、距离、时长、获得能量
- 点击查看详细信息
- 支持筛选（按日期、按区域、按距离）
- 支持搜索

**数据统计可视化**：

**今日数据**：
- 今日骑行次数
- 今日总里程
- 今日总时长
- 今日获得能量
- 今日减排量

**本周数据**：
- 每日骑行里程柱状图
- 每日骑行时长折线图
- 本周总里程、总时长
- 周目标完成进度

**本月数据**：
- 月度骑行日历（热力图）
- 每周里程对比
- 月度总里程、总时长
- 月度排名变化

**累计数据**：
- 累计总里程（大数字展示）
- 累计骑行次数
- 累计骑行时长
- 累计减排量（换算成种树数量）
- 累计获得能量
- 里程碑进度（如距离下一个100km还有多少）

**数据对比**：
- 与上周/上月对比
- 与好友对比
- 与城市平均对比
- 个人最佳记录

**可视化图表**：
- 里程趋势图（折线图）
- 骑行时段分布（饼图）
- 区域分布（地图热力图）
- 月度日历热力图
- 速度分布图

**个人报告**：
- 周报：本周骑行总结、亮点数据、排名变化
- 月报：月度骑行分析、成就回顾、下月目标建议
- 年报：年度骑行大数据、个人标签、精彩瞬间

**设计要点**：
- 数据记录准确完整
- 可视化清晰美观，使用图表库
- 数据对比增强成就感
- 个人报告有仪式感
- 支持数据导出和分享

#### 【核心层】探索者成长系统

这是整个游戏化系统的核心，所有其他功能都围绕这个成长线展开。

##### 3.3.1 探索者等级系统

**功能描述**：
用户作为"城市探索者"，通过骑行获得经验值，提升等级（1-100级）。

**经验值获取**：
- 骑行距离：1km = 10经验值
- 骑行时长：10分钟 = 5经验值
- 解锁新区域：+100经验值
- 完成路线：+50-200经验值（根据难度）
- 参与挑战赛：+200-1000经验值
- 分享路线被点赞：+10经验值/赞

**等级权益**：
- 每10级解锁新的伙伴种类
- 每5级解锁新的技能
- 高等级获得专属称号和头像框
- 等级越高，能量获取效率越高（最高×1.5）

**等级划分**：
- 1-10级：新手探索者
- 11-30级：城市漫游者
- 31-50级：资深骑士
- 51-70级：探索大师
- 71-90级：传奇骑士
- 91-100级：城市守护者

**设计要点**：
- 等级是长期目标，给用户持续动力
- 等级权益实用且有吸引力
- 升级曲线合理，避免后期过于困难

##### 3.3.2 骑行伙伴养成系统

**功能描述**：
用户可以培养专属的虚拟骑行伙伴，伙伴陪伴用户骑行并共同成长。

**伙伴设定**：
- 初始伙伴：美团小袋鼠（品牌IP）
- 可解锁伙伴：城市系列（熊猫、麻雀、猫咪等）、神话系列（青龙、白虎等）
- 每个伙伴有独特外观和技能

**养成机制**：
- 使用能量喂养伙伴，伙伴获得经验值
- 伙伴等级：1-50级
- 每次骑行伙伴陪伴，增加亲密度
- 亲密度达到一定值解锁伙伴故事和特殊技能

**伙伴技能**：
- 能量加成：提升能量获取效率（+10%-30%）
- 经验加成：提升经验值获取（+10%-20%）
- 幸运加成：增加稀有奖励概率
- 探索加成：解锁新区域时额外奖励

**伙伴进化**：
- 达到10级、25级、50级时可进化
- 进化后外观变化，技能增强
- 进化需要消耗能量和特殊材料（通过挑战赛获得）

**设计要点**：
- 伙伴是情感连接的核心，增强用户粘性
- 养成过程有成就感
- 多样化的伙伴满足不同用户喜好
- 伙伴技能实用，影响游戏体验

##### 3.3.3 虚拟单车养成系统

**功能描述**：
用户可以培养和升级虚拟共享单车，解锁不同的单车皮肤和性能。

**单车设定**：
- 初始单车：经典美团单车（黄色）
- 可解锁单车：城市系列、科技系列、复古系列、限定系列
- 每辆单车有独特外观和属性加成

**单车系列**：

**城市系列**（10级解锁）：
- 北京单车：红墙黄瓦配色，能量+5%
- 上海单车：摩登都市配色，经验+5%
- 成都单车：熊猫主题配色，星币+5%

**科技系列**（25级解锁）：
- 赛博单车：科幻风格，能量+10%
- 极光单车：炫彩光效，经验+10%
- 量子单车：未来感设计，星币+10%

**复古系列**（40级解锁）：
- 民国单车：复古造型，能量+12%
- 蒸汽单车：工业风格，经验+12%
- 老爷车单车：经典设计，星币+12%

**限定系列**（特殊获得）：
- 春日樱花单车：春季挑战赛奖励
- 夏日海洋单车：夏季挑战赛奖励
- 黑金单车：黑金会员专属

**养成机制**：
- 使用能量升级单车，提升属性
- 单车等级：1-30级
- 每次骑行使用当前单车，增加熟练度
- 熟练度达到一定值解锁单车特性

**单车属性**：
- 能量加成：提升能量获取效率（+5%-15%）
- 经验加成：提升经验值获取（+5%-15%）
- 星币加成：提升星币获取（+5%-15%）
- 特殊效果：骑行轨迹特效、音效等

**单车升级**：
- 1-10级：每级消耗50能量
- 11-20级：每级消耗100能量
- 21-30级：每级消耗200能量
- 升级后属性提升，外观更炫酷

**单车切换**：
- 可随时切换使用的单车
- 不同单车适合不同场景
- 收集所有单车获得"单车收藏家"称号

**设计要点**：
- 单车养成与伙伴养成并行，双重乐趣
- 单车皮肤视觉差异明显，有收集价值
- 属性加成实用，影响游戏体验
- 限定单车增强稀缺性和成就感
- 与伙伴系统形成互补，不冲突

##### 3.3.3 能量与星币双货币系统

**功能描述**：
引入双货币系统，分离养成和兑换功能，确保用户既能享受养成乐趣，也能获得实际优惠。

**能量系统（用于养成）**：

**能量获取**：
- 基础骑行：1km = 10能量
- 时长奖励：10分钟 = 5能量
- 连续骑行：连续3天 +20%，7天 +50%，30天 +100%
- 规范停车：+10能量
- 解锁新区域：+50能量
- 完成路线：+30-100能量
- 挑战赛奖励：+100-1000能量
- 伙伴技能加成：+10%-30%
- 探索者等级加成：最高×1.5
- 美团会员加成：黄金×1.1，铂金×1.2，黑金×1.3，黑钻×1.5

**能量用途（仅用于养成）**：
- 喂养伙伴：20-100能量/次
- 伙伴进化：500-2000能量
- 升级单车：50-200能量/次
- 单车进化：300-1500能量
- 解锁技能：100-500能量
- 购买养成道具：50-500能量

**星币系统（用于兑换优惠券）**：

**星币获取**：
- 完成每日任务：+10星币
- 完成每周任务：+50星币
- 完成挑战赛：+100-500星币
- 探索者升级：每级+20星币
- 伙伴升级：每级+10星币
- 单车升级：每级+10星币
- 分享路线被完成：+5星币/次
- 创作优质内容：+50-200星币
- 连续签到：第7天+30星币

**星币用途（仅用于兑换优惠券）**：
- 外卖券：50星币 = 5元，100星币 = 10元
- 酒店券：200星币 = 30元
- 电影券：80星币 = 10元
- 通用神券：150星币 = 15元
- 骑行卡：100星币 = 7天免费骑行卡

**双货币可视化**：
- 能量：闪电图标⚡，美团黄色
- 星币：星星图标⭐，金色
- 顶部同时显示两种货币余额
- 不同页面突出显示对应货币

**设计要点**：
- 双货币分离养成和兑换，避免冲突
- 能量主要通过骑行获得，鼓励多骑行
- 星币主要通过完成任务和成就获得，鼓励多样化行为
- 两种货币都有价值，用户不会只关注一种
- 星币获取相对较慢，保持优惠券的价值感

##### 3.3.4 技能系统（等级解锁）

**功能描述**：
探索者达到指定等级后自动解锁技能，提升骑行体验和奖励效率。

**技能列表**：

**效率技能**（自动解锁）：
- 5级：能量加成I（能量获取+10%）
- 15级：能量加成II（能量获取+20%）
- 30级：能量加成III（能量获取+30%）
- 10级：经验加速I（经验值获取+10%）
- 25级：经验加速II（经验值获取+20%）
- 20级：幸运之星（稀有奖励概率提升）

**社交技能**（自动解锁）：
- 8级：魅力光环（分享路线获得更多点赞）
- 18级：团队领袖（组队骑行额外奖励+20%）
- 35级：导师（邀请好友获得更多奖励）

**特殊技能**（自动解锁）：
- 40级：伙伴加速（伙伴升级所需能量-20%）
- 50级：神券优惠（能量兑换神券8折）
- 60级：双倍日（每周1天能量获取×2）

**设计要点**：
- 技能到达等级自动解锁，无需手动激活
- 技能提供实际游戏体验提升
- 不设置复杂的技能树，保持简洁
- 技能效果清晰可感知

#### 【探索层】城市探索系统

**区域划分**：

**区域功能**：
- 查看区域内的特色路线

**功能描述**：

- **文化路线**：历史建筑、博物馆、艺术区

**路线标签**：
- 难度：轻松、适中、挑战
- 距离：<3km、3-5km、5-10km、>10km

**设计要点**：
- 路线多样化，满足不同需求
- 标签清晰，方便筛选
- 路线质量有保证
- 不直接推荐商家，但路线可能经过美食区域

##### 3.4.3 城市记忆收集

#### 【生态层】美团生态联动

将美团生态优势融入游戏化系统，但不过度商业化。

##### 3.8.1 星币兑换神券

**功能描述**：
星币可以兑换美团神券，作为额外福利，与养成系统分离。

**兑换选项**：
- 外卖券：50星币 = 5元，100星币 = 10元
- 酒店券：200星币 = 30元
- 电影券：80星币 = 10元
- 通用神券：150星币 = 15元
- 骑行卡：100星币 = 7天免费骑行卡

**兑换原则**：
- 使用星币而非能量，避免与养成冲突
- 星币获取相对较慢，保持优惠券价值
- 用户可以同时享受养成和兑换优惠
- 兑换比例合理，不贬值星币

**设计要点**：
- 星币专用于兑换，能量专用于养成
- 双货币系统确保用户两种乐趣都能享受
- 不过度强调兑换，避免过度商业化
- 保持星币在游戏内的价值感

##### 3.8.2 会员成长加速

**功能描述**：
骑行行为可以加速美团会员等级提升。

**联动机制**：
- 每获得100能量 = 1点会员成长值
- 完成季度挑战：额外+50成长值
- 探索者等级提升：额外成长值奖励
- 月度骑行达标（20次）：+100成长值

**会员反哺**：
- 黄金会员：能量获取×1.1
- 铂金会员：能量获取×1.2 + 每月1张免费骑行卡
- 黑金会员：能量获取×1.3 + 每月2张免费骑行卡
- 黑钻会员：能量获取×1.5 + 每月5张免费骑行卡 + 神券兑换8折

**设计要点**：
- 打通骑行与会员体系
- 形成正向循环
- 会员权益实用
- 不强制用户关注会员

##### 3.8.3 特色路线区域商家信息

**功能描述**：

- 路线详情页显示"沿途区域"
- 标注美食区、咖啡区等
- 用户可以点击查看该区域的商家列表
- 不主动推送，用户主动查看

**信息内容**：
- 区域内美团商家列表
- 商家评分和特色
- 距离路线的位置
- 用户可以收藏感兴趣的商家

**设计要点**：
- 不直接推荐商家，避免打扰
- 以区域形式呈现，更自然
- 用户主动查看，不强制
- 为商家引流但不过度商业化

## 四、用户体验设计原则

### 4.1 完整性原则
- 所有功能围绕"探索-成长-分享"核心
- 统一的世界观和视觉风格
- 能量是唯一货币，避免系统复杂化
- 各系统相互联动，形成闭环

### 4.2 渐进式成长原则
- 新手期快速获得成就感（伙伴、首个勋章）
- 中期多样化目标（路线探索、挑战赛）
- 长期稀有奖励（传奇称号、稀有伙伴）
- 成长曲线平滑，避免断层

### 4.3 情感连接原则
- 伙伴养成建立情感纽带
- 城市探索增强地域认同
- 社区分享形成归属感
- 温暖的文案和视觉设计

### 4.4 社交适度原则
- 提供社交功能但不强制
- 竞争机制温和，避免焦虑
- 强调个人成长而非排名
- 保护用户隐私

### 4.5 生态融合但不喧宾夺主原则
- 美团生态是额外福利，不是核心
- 游戏化体验优先，商业化适度
- 不过度推荐商家
- 保持产品的纯粹性

- ROI验证 → 小范围试点后再推广
